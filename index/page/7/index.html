Win10 Hyper-V 和 VirtualBox 冲突的问题

该问题的本质是：**VirtualBox 和 VMWare Workstation 为什么不能在 Hyper-V 里运行**。

参考：

  • https://blog.csdn.net/Dream_Weave/article/details/105730994
  • https://superuser.com/questions/1208850/why-cant-virtualbox-or-vmware-run-with-hyper-v-enabled-on-windows-10

在 windows 10 上运行的话，可以这么理解：virtualbox 依赖于 windows 10，而在 windows 10 上安装的 docker 所需要的 Hyper-V，居然是 windows 10 所依赖的！也就是说开启了 Hyper-V 之后的 windows 10 类似于在 Hyper-V 上通过虚拟化安装运行了 windows 10，这个我不是很能理解。

总之，所谓 Hyper-V 和 VirtualBox 冲突的问题应该转换成：“VirtualBox 和 VMWare Workstation 为什么不能在 Hyper-V 里运行”。

转换后的这个问题可以这么回答：在虚拟机中，Intel 的 VT-X 指令就不再可用了，所以，在 Hyper-V 安装了的情况下，VirtualBox 和 VMWare Workstation 本身作为虚拟机，是不能够再使用 VT-X 指令了，所以也就不能够正常运行。
