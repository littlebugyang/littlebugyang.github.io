Chapter_1_Getting_Started_with_C++

关于 C++ 的特性

C++ 的三种编程类别的集大成者：

  1. 以 C 语言为代表的面向过程编程（procedural programming）
  2. 以类（class）为代表的面向对象编程（object-oriented programming）
  3. 以 C++ 模板（template）为代表的泛式编程（generic programming）

C++ 的发展历史

在 C++ 之前

更笼统地说，在高级编程语言诞生之前，程序员们使用的是汇编语言（assembly language）。汇编语言是特定于电脑处理器（CPU）的，也就是说，要让汇编语言写成的程序，在另一台用着不同处理器的电脑上运行，程序需要重新编写，而且，是用另外一种汇编语言。

于是为了提高程序的可移植性，专家们研究出了高级编程语言。其中，Ritchie 结合了低级语言和高级语言的特性，集合了硬件高效访问、普遍性和可移植性等特点，创造出了 C 语言。

C 语言是面向过程的，这意味着 C 语言着重程序的算法（algorithm）端，而算法则是程序使用的方法。算法与数据一起，构成了程序：Data + Algorithms = Program。

而科学家们还建立了一种编程风格：结构化编程（structured programming），主要用于避免程序员们写出难以理解的程序。

接着，结构化编程衍生出了自顶向下（top-down）的设计模式，C 语言提供了许多特性用以辅助用户进行结构化编程，当然也有助于实现自顶向下，具体的实现方法是：把任务分解成更小的、更可控的子任务。

C++ 的诞生

与 C 语言一样，C++ 诞生于贝尔实验室。20世纪80年代，Bjarne Stroustrup 在这里研发出了 C++ 编程语言。

与 C 语言的着重算法、面向过程不同，面向对象编程着重于数据，而 C++ 正是面向对象编程的典型编程语言。

尽管 C++ 是面向对象编程的程序语言，强调数据的重要性，但是 C++ 的另一重大特性：泛式编程，却强调对某种特定数据类型的脱离。

C++ 是 C 的超集，也就是说任何有效的 C 程序同时也是有效的 C++ 程序。C++ 程序可以使用 C 的软件库。C++ 与 C 的这两层关系，大大提高了 C++ 在世界范围内的传播速度。

C++ 的名字来源于 C 中的自增操作符 ++。

C++ 的统一

移植程序的第一道门槛是硬件，而第二道门槛则是编程语言的不统一。

第二道门槛并不是皆使用同一种编程语言，如：C++，即可解决。此处的语言不统一，指的是 C++ 的使用群体内部没有达成统一的 C++ 使用标准。

为了能够尽量让大家所使用的 C++ 是相互兼容的，相关组织参与到 C++ 的标准制定中：

  1. ANSI（American National Standards Institute）在 1990 年的时候创建了专门制定 C++ 标准的委员会。在此前，ANSI 已经创建过 C 语言的标准了。ISO（International Organization for Standardization）不久后也加入到给 C++ 制定标准的过程中。
  2. 1998 年，在经过数年的努力后，关于 C++ 的国际标准出炉，ISO，IEC（International Electrotechnical Commision）和 ANSI 等都采用了这个标准。而人们也通常称这个标准为 C++98。该标准不仅总结了当时 C++ 的特性，而且还加入异常（exception）、运行时类型识别（runtime type identification, RTTI）、模板（template），还有标准模板库（standard template library, STL）。
  3. 2003 年，C++ 标准的新版本出台，但这版本只是在 C++98 的基础上对其进行了一些修改，该标准被称作 C++03。由于 C++03 并没有更改语言特性，所以人们常用 C++98 指代 C++98/C++03。
  4. 2011年，ISO 委员会推出 C++11 标准。该标准如 C++98 那般给这门编程语言添加了许多新特性。该标准的名字本应为 C++03 之后的 C++0x，其中 x 本应为 7 或 8，但是编程语言标准制订这件事，实在是太过于耗时耗力了，以至于人们只能把“0x”解释为 “16 进制整数”，从而可最大延伸至数字“15”。这样一来，标准的出台期限就可以拖到 2015 年。所以，从这个角度看，C++11在 2016 年之前出台，可看作是提前完成了目标。
  5. ISO 的 C++ 标准制订工作，同时推动了 ANSI C 的标准发展，因为 C++ 本来就被看作是 C 的超集——在理想状态下，任何有效的 C 程序都应该是有效的 C++ 程序。事实上，ANSI C 的确吸收了一些首先出现在 C++ 中的特性，比如说函数原型（function prototype），还有常量限定符（const type qualifier）。
  6. 在 ANSI C 出台之前，社区已经出现了一个事实上的标准，其基于 Kernighan 和 Ritchie 的著作：“The C Programming Language”。这个标准也被冠以“K&R C”的名字。在 ANSI C 出台之后，“K&R C”也被称作“经典 C”。
  7. ANSI C 不仅定义了 C 语言，而且定义了标准 C 语言库，C++ 也使用了这个库。而且，ISO C++ 也提供了基于 C++ 类的标准库。
  8. 上一个 C 语言的标准库是 C99。1999 年，ISO 采用了该标准，1 年后 ANSI 也采用了。

创建程序的机制

从源码到可执行程序

在不同电脑环境和使用不同编译器（compiler）的情况下，从源码到可执行程序的过程大同小异。暂不论小异，大同在于它们都应该遵循以下步骤：

  1. 创建源文件：使用文本编辑器编写源代码（source code），保存为源文件。
  2. 编译源代码：使用编译器，将源文件中的源代码翻译成中间语言，这里的中间语言是机器语言（machine language），这是可以被主机直接使用的。这一过程生成的翻译结果是本程序的目标代码（object code）。
  3. 链接目标代码：把目标代码与其他的代码链接起来，生成最终的可执行代码（executable code）。“其他代码”包括库代码（library code）和启动代码（startup code）。其中库代码是程序所使用到的 C++ 函数库中各类函数的目标代码；而启动代码是一小块为程序执行作准备的汇编代码。

源码扩展名

包含源代码的文件是源文件。源文件名的后缀很重要，恰当的后缀名可以提高可阅读性，而更重要的是，让编译器也知道这是一个 C++ 的源文件。

相对于 Windows 的大小写不敏感，Unix 系统是大小写敏感的，所以，在 Unix 系统下可以使用大写的“C”后缀来表示 C++ 源文件：main.C，同时，使用小写的“c”来表示 C 语言源文件：main.c。在其他一些 Unix 系统下，可以使用“cc”、“cxx”等后缀来表示 C++ 源文件。

不同环境下的具体创建程序方法

Stroustrup在实现 C++ 的时候，并没有直接开发 C++-to-object code 的编译器，而是使用 C++-to-C 的编译器：cfront（for C front end），将 C++ 源码转化为 C 语言的源码，再将 C 语言的源码通过标准 C 语言编译器。虽然这种方法有点迂回，但是大大提高了 C++ 在 C 语言社区的传播速度。

Linux

Linux 操作系统大多数使用 g++，这是一款来自自由软件基金会（Free Software Foundation）的 GNU C++ 编译器。

g++ 编译器的使用方法与标准的 Unix 编译器非常相似。

Command-Line Compilers for Windows Command Prompt Mode

在 Windows 的命令行模式下，可以使用包含 GNU C++ 编译器的安装包，如 Cygwin 和 MinGW（Minimalist GNU for Windows），编译器的名字也为 g++。

Windows

在 Windows 的桌面环境下使用 IDE 可以给开发者提供更多选择：Compile，Build，Make，Build All，Link，Execute，Run，还有 Debug。

  1. Compile：编译当前打开的源文件。

  2. Build or Make：编译项目文件里的所有源文件。这往往是一个增量过程，也就是说，在只改变部分文件的情况下，再次进行 Build or Make 只会对已修改文件重新编译。

  3. Build All：通常指将所有源文件，从头开始进行编译。

  4. Link：将编译过的代码与必要的库代码结合。

  5. Run or Execute：运行程序。如果在运行程序之前，没有经过编译、连接等过程，则 IDE 会自动执行。

  6. Debug：调试。提供可逐步运行程序的选项。在调试时，应该首先解决第一个出现的错误，因为后续的错误可能皆是由第一个错误引起的。

  7. Release：有些编译器还会提供 Debug 和 Release 两种不同版本的可执行文件。Debug 版本的程序相较于 Release 版本添加了额外代码，提供更为细化的调试特性，这是以更大的程序体积和更慢的执行速度为代价的。

Macintosh

在 Mac OS X 操作系统上可以使用 Xcode 进行开发。

Xcode 不仅提供了支持数种编程语言的 IDE，而且安装了一些编译器：g++ 和 clang。

其他

需要注意的是：某个编译器成功通过了某个程序的源文件，并不代表着该程序的源文件是有效的，比如说，该源文件使用了一些已经过时的 C++ 特性；反过来，某个编译器没有通过某程序的源文件，也不代表着该程序的源文件是无效的，比如说，该源文件使用了一些未被编译器实现的特性。
