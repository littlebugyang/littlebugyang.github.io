使用包含，还是私有继承

私有继承相较于的包含缺点：

  1. 私有继承的对象匿名，程序可读性不强，使得类间关系看起来抽象不少
  2. 私有继承通常要多重继承，问题很容易发生，比如两个基类的方法名相同，又比如两个基类有共同祖先
  3. 私有继承继承一个基类，只对应增加一个匿名子对象

但是有一些场景，还是需要私有继承才能做到：

  1. 某类有受保护成员，现通过包含，使该类成为新类的成员，则无法新类无法访问某类成员的受保护成员；若通过私有继承，则新类可以访问某类的受保护成员。
  2. 重定义虚拟函数。通过包含，包含类无法重定义被包含类的虚拟函数，因为虚拟函数的重定义是专供于派生类的。私有继承下，重定义后的虚拟函数也只能在类内部使用。

受保护继承

受保护继承是私有继承的变种。

受保护继承下，基类的公共和受保护成员都变成派生类的受保护成员。跟私有继承一样，基类的接口仅对派生类可用，而不对外部世界可用。

受保护继承与私有继承的主要区别在于派生类再派生时：私有继承下，再派生类无法访问基类接口，因为已经被派生类私有化了；而受保护继承下，再派生类还是可以访问基类接口的，毕竟派生类只是把其当作受保护成员，依旧可供派生类使用。

虚拟基和非虚拟基的混合

当某类通过不同的路径继承了一个基类时，某类实例化后的对象中：

  • 对所有的虚拟路径，拥有一个基类对象
  • 对每条非虚拟路径，都各拥有一个基类对象

举例：B 是 C 和 D 的虚拟基类，B 是 X 和 Y 的非虚拟基类。假设 M 继承了 C、D、X、Y，则由 M 到 B 有 4 条路，其中 2 条为经过 C 和 D 的虚拟路径，另外 2 条是经过 X 和 Y 非虚拟路径。最后 M 因为 C 和 D 拥有 1 个 B 类对象，因为 X 和 Y 拥有 2 个 B 类对象。

虚拟基类和支配

在类继承，尤其是多重继承时，经常会出现函数同名的场景。此时，如果其中一个函数支配了其他函数，则该函数的调用不需要任何限定符，就能不带歧义地执行。

派生类中的函数名，支配其任何祖先内的同名。

虚拟歧义规则无视访问规则。即便方法在其中一个祖先处是私有的，且在另一个祖先处有同名方法，则在该派生类中使用该方法还是产生歧义的。相似地，尽管当前直接祖先的同名方法为私有，且间接祖先的同名方法为公有，在派生类中直接调用同名方法，会被视作非法调用私有方法。

定义类模板

定义模板类的语法：

新的 C++ 实现推荐使用 typename 而不使用 class，毕竟 Type 不是一定得要是类。

对于一个现存的类方法，将其改造成模板类，需要如下步骤：

  • 每个函数前面都加上同样的模板宣言：template <typename Type>
  • 把用 typedef 创建出来的标识符，换成 Type
  • 在类名和作用域解析符之间添加 ，比如，Stack<Type>::。
  • 如果方法是在类声明中定义的，也这是行内定义，此时可以省去模板头和类限定符。

需要注意的是：模板不是类和成员函数定义，而是指导编译器生成类和成员定义的准则。任何模板都必须经过实例化（instantiation or specialization）后才能被使用。

因为模板成员函数并不是真正的函数，所以把它们放在一个单独的实现文件中并不会使模板生效，它们不可以被分别编译，必须要把某个模板的所有的信息放在同一个头文件里。

表达式参数有所限制：
