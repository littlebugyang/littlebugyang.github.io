Chapter_3_Dealing_with_Data

面向对象编程（object-oriented programming，OOP）的本质是：设计并扩展用户自定义的数据类型。

C++ 内置的变量类型分为两种：基础类型和复合类型。这一章主要讲基础类型的变量

变量

变量名

给变量命名需要遵守以下的命名规则：

  • 名字只能为字母、数字和下划线的组合。
  • 首字符不能为数字。
  • 大小写敏感。
  • 不能使用 C++ 关键字。
  • 不能以“双下划线”（__）开头，也不能以“单下划线+大写字母”（_[A-Z]）开头，要预留给编译器及其使用的资源。若否，编译器也不会报错，但是后果不可预知。
  • 其他以“单下划线”（_[a-z0-9]）开头的名字用于全局标识（global identifier）。
  • C++ 对名字无长度限制，但是某些平台对名字有限制。

整型类型

整型宽度

用于描述整型类型所占用的内存大小的术语是：宽度。

宽度从小到大排列，有如下整型类型：

char, short, int, long, long long(C++11).

每个类型都有 signed 和 unsigned 版本。

在不同的机器上，各种整型类型的宽度都不尽相同。但是 C++ 规定了最小宽度和一些其他的规则：

  • short 最少 16 位宽
  • int 至少和 short 一样宽
  • long 最少 32 位宽，至少和 int 一样宽
  • long long 最少 64 位宽，至少和 long 一样宽

查看特定环境下某类型的宽度，可以使用以下两种方法：

无符号类型

类型 unsigned 是 unsigned int 的缩写。

因为同宽度的有符号类型和无符号类型代表的内容不同，所以 C++ 无法保证有符号类型和无符号类型之间能够正确无误地相互转化。

整型类型的选择

通常地，对计算机来说，最为自然的整型类型是 int。自然是指计算机处理得最有效率。

选择整型类型时，可以遵守以下准则：

  • 在没有强制要求的情况下，尽量使用 int。
  • 如果数据从来都是非负的，那么可以使用 unsigned int。
  • 如果已知数据会超过 16 位的整型数，那么就应该使用 long，尽管系统上的 int 为 32 位。这是为了将来把你的程序迁移至 int 为 16 位的系统时，程序不至于因为整型数的范围变窄而无法运转。
  • 如果 short 的宽度比 int 小的话，使用 short 可以节省一些内存，尤其是要使用整型数组的时候。

整型文本值

文本值（literal），或者说常量（constant），是明确写在源代码上的值。

在源代码中，可以以各种进制书写整型数值：

  • 十进制：默认进制，比如，int x = 1;。
  • 八进制：在文本值前面加数字“0”，比如：int octal_x = 042;。
  • 十六进制：在文本值前面加数字字母“0x”或“0X”，比如：int hex_x = 0x2a;。

无论是哪种进制形式，在电脑的存储介质中，以上的变量都是以二进制的形式留存的。

使用 cout 输出以上初始化的变量时，输出的形式默认是十进制（decimal）。

整型文本值的类型

C++ 默认把整型常量存储为 int 类型，除非在以下任一情况：

  • 常量的后缀明确了类型（大小写皆可）：
    • 常量后缀为一个字母（L），表示整型常量为 long
      • 如：long x = 1234L;。
    • 常量后缀为两个字母（L），表示整型常量为 long long
      • 如：long long x = 12345Ll;。
    • 常量后缀为三个字母（两 L 一 U），表示整型常量为 unsigned long long
      • 如：unsigned long long x = 12341ulL;。
  • int 的取值范围不够容下该常量：
    • 对于一个不带后缀的十进制整型常量，存储其的类型为 int、long、long long 中能容下它的最小者。
    • 一个不带后缀的八进制或十六进制整型常量，其代表的类型为 int、unsigned int、long、unsigned long、long long、unsigned long long 中能容下它的最小者。
    • 举个例子，一个把十进制的 40000 认定为 long 类型的电脑系统，在面对同样值但是却不同形式的常量 0x9C40 时，会将其看作 unsigned int 类型。把 40000 认定为 long，说明 int 的取值范围为 [-32768, 32767]，int 为 16 位。但是，十六进制的常量通常用来代表内存地址，而内存地址是无符号的，所以这里可以使用 unsigned int，也就是最大值可达 65535，足以容下 40000。

char 类型

char 类型是用来存储字符的。因为编程语言用数字给字符编码，所以 char 实际上也是一种整型类型。

char 的范围足够容纳基本符号：所有的字母、数字还有标点符号等等。

char 的大小是一字节，也就是可以代表 128 个数字，即便是在一些仅支持少于 128 种字符的系统上，也是足够了。

当我们使用 cin 输入到 char 类型的变量时，实际存储的是整型数。但是在使用 cout 输出该变量时，显示的却是整型数编码对应的字符。该转换的关键在于 cin/cout 上，而不在 char 本身。

cout 根据变量类型决定如何输出该变量的值；而对于 cin 来说，同样地，也根据变量类型选择如何处理输入到变量的值。比如说，尽管输入的是数字，但只要是输入到 char 变量，cin 也会将其变成数字字符所对应的编码，然后存入到变量中。但究其本质来讲，cin 将输入的内容都看作是字符，输入到 char 变量中，实际上是最正常的行为；而输入到 int 等类型时，cin 则要分析输入的字符，将其转化为整型数值，再将该值存入变量中。

在美国，最常见的符号集是 ASCII 字符集。但是 C++ 中并不一定采用该字符集，而是取决于宿主机，比如说，在 IBM 大型机上，就使用了 EBCDIC 字符集。

cout.put()

cout.put() 可以输出一个单字符。put() 是对象 cout 的成员函数，. 是成员操作符。

仅仅使用 cout 就可以输出单字符，没必要单独设立这个成员函数，其存在是一个历史遗留问题：在 C++ 2.0 之前，C++ 把诸如 'M'、'n' 这样的字符常量看作是 int 类型，所以在 cout << '$'; 的时候就会像输出 int 类型的变量一样，输出该字符在字符集中的编码，所以需要 cout.put('$'); 在屏幕上显示字符。

wchart_t

程序员可能需要处理更大的字符集，大到单个 8 比特的字节无法容纳的字符集，C++ 可以有几种解决方法：

  • 第一种方法是直接扩大 char，如果这么大的字符集，都是环境所必需的话，编译器可以把 char 类型定义成 16 位宽，甚至更宽。
  • 第二种方法是令小字符集和大字符集同时存在，用通常 8 位宽的 char 来表示基础的小字符集，而用另外一种类型：wchar_t，来表示扩展后的字符集。
    • wchar_t 的含义是：“wide character type”。
    • wchar_t 本质上也是整型类型，其足够宽以容下扩展后的字符集。
    • wchar_t 与其他的整型类型中的某型，有着相同的宽度和符号属性，某型被称作基础类型（underlying type），在不同的环境下，基础类型的选择也是不同的。
    • iostream 头文件提供了设施：wcin 和 wcout ，用以处理 wchar_t 流。
    • 表示一个 wchar_t 的字符常量，需要在常量前面加上 L 前缀，比如：L'test'。

C++11 新类型：char16_t 和 char32_t

随着编程社区对 Unicode 的使用经验逐渐积累，人们发现，wchar_t 也开始不够用了。其原因是：在计算机系统上编码字符和字符串是一件十分复杂的事情，仅仅使用 Unicode 码点，还不足以解决问题。

在编码字符串的时候，使用大小和符号确定的类型，是十分有用的。但是 wchar_t 的符号和大小在不同的编程环境下却是不同的。因此 C++11 引入了 char16_t 和 char32_t，两者都是无符号的，而且如其名，char16_t 占 16 位宽，char32_t 占 32 位宽。

跟 wchar_t 一样，char16_t 和 char32_t 都是有内置的整型类型的，也就是都有基础类型，而这个基础类型也是根据环境的不同而有所差别的。但是与 wchar_t 不同的是，char16_t 和 char32_t 的大小和符号是确定的。

浮点数

浮点数（floating-point numbers），是有分数部分的数。计算机存储浮点数的方式是：一部分代表值，其他部分负责调整值的量级。打个比方，34.1245 和 34124.5，它们在计算机中，值是一样的，但是量级不一致。

浮点数的表示范围可以很大，也可以很小。

浮点数表示方法

  1. 标准的小数点标记法
    • 尽管小数部分为 0，只要有小数点，该值就应该以浮点数的形式存储起来，比如，9.0 应该为浮点类型。
    • C++ 标准允许不同环境下的不同表示量级的方法，比如说，欧洲方法中使用逗号 , 来表示小数点。但是只需要在输入和输出的时候，选择不同的表示方法即可，这并不会对数值本身造成影响。
  2. E 标记法
    • 比如说，3.45E6，意为着：$3.45\times 10^6$。E 标记前面的部分是尾数（mantissa），E 标记后面的部分是指数（exponent）。
    • E 标记不区分大小写，尾数和指数都可以是负数，正数时可选择添加正号。
    • 尾数、E 标记、指数三者之间不允许存在空白
    • 尾数部分为整数，没有小数点时，该带 E 标记的数值还是以浮点数的格式存储。
    • E 标记法的 d.dddE+n，这意味着，d.ddd 的小数点要往右移 n 位；相反地，d.dddE-n 意味着，d.ddd 的小数点要往左移 n 位。移动的小数点是浮点数中的“浮点”。

浮点数类型

C++ 有三种浮点数类型：float、double 和 long double。

这三种浮点数在“可表示有效数字位数（significant figures）”和“最小指数允许范围”两方面有所区别。

有效数字是数字中有意义的数字的位数：

  • 形式上看，从数字最右边不为 0 的数字开始，包括该数自身，直至数字最左端，该区间内数字的个数为有效数字的位数
  • 本质上看，能完全地用尾数和指数表示该数字后，抹去尾数末端的零，尾数剩下的数字个数即为有效数字个数。

C 和 C++ 要求：

  • float 类型值的有效数字位数，至少为 32 位
  • double 类型值的有效数字位数，至少为 48 位，而且必须比 float 多
  • long double 类型值的有效数字位数，必须要比 double 多

很明显，以上三者可以为同一大小。实际上，通常情况下，float 为 32 位，double 为 64 位，long double 为 80、96、或者 128 位。而且，三种类型的指数范围最小为 [-37, 37] 。可以 cfloat 或者 float.h 头文件中找到当前系统的限制：

计算机中，浮点数运算后得到的结果不一定是正确的。

浮点数常量

默认状态下，浮点数常量都是 double，比如说 8.24，2.4E8 等。

  • 如果想要表达 float 类型的浮点数常量，需要在常量后加上 f 或者 F 的后缀；
  • 如果想要表达 long double 类型的浮点常量，需要加上 l 或者 L 的后缀，因为 l 看起来跟数字 1 很像，所以用大写的 L 更好。

浮点数的优缺点

优点：

  1. 可以表示整型数之间的数
  2. 可以表示更大范围内的数

缺点：

  1. 处理速度慢
  2. 精度可能丢失。比如说：
    • 表示无限小数时，精度一定丢失
    • 操作影响很小，小到不足以影响有效数字，导致不发生变化，精度丢失
    • 即便是很小很简单的小数，也不能够用有限的位数完整表示

布尔类型

布尔类型变量值可以为 true 或者 false。

C++ 把非零值译成 true，而把零译成 false。

true 和 false 可以通过变量提升而成为 int 类型，分别成为 1 和 0。

任何数值或者指针值都可以隐式转换为 bool 值。

auto 类型

C++11 引入 auto 类型，允许编译器从一个有初始值的变量中推断出变量类型。

C++ 重定义了 C 中的 auto 关键字，这个关键字先前很少被使用。

const 限定符

使用 const 限定符初始化变量后，编译器不再允许修改该变量的值。

const 被称作限定符（qualifier），因为它限定了变量声明的含义。

为了提醒开发者该变量被 const 限定了，通常会有以下做法：

  • 变量的名字以大写字母开头
  • 变量的名字全部都是大写字母
  • 变量名字以 k 开头，比如说：kmonths

在声明 const 变量的时候，最好同时赋值，否则，语句会创建一个变量，而其值不确定且不可修改。

与 #define 指令相比，const 限定符要好很多，因为 const 限定符还有其附带的一系列范围规则（scope rules），可以让我们更灵活地使用常量，而且，const 还可以规定常量的类型。

ANSI C 中也有 const 限定符，但是与 C++ 中的 const 限定符有点区别。首先是范围规则的区别，其次是 C++ 中可以使用 const 值来声明数组的大小。

C++ 算术运算符

C++ 中有五种基础的算术：加减乘除模。每个操作都需要两个操作数（operand）。

运算符和操作数组成了表达式（ expression）。

运算符顺序：运算符优先级和关联性

当多个运算符可以应用到同一个操作数的时候，C++ 利用优先级规则（precedence rules）决定使用哪一个运算符。

两个运算符拥有同样优先级时，C++ 会根据关联性作出决定。关联性有自左向右（left-to-right）和自右向左（right-to-left）两种。

自左向右关联意味着，如果两个运算符在同一个运算数有着同样的优先级时，首先应用左边的优先级；自右向左关联则相反。

人在直觉上会觉得左边的乘法会先于右边的乘法得出结果，但是这两个乘号并不应用在同一个操作数上，所以关联性不起作用。C++ 把这个问题留给了具体的实现环境，让环境决定在该系统上的最优选择。

除法转移

除法运算的效果决定于操作数的类型。

除法运算时，若两个操作数都是整型数，则结果的小数部分会被舍弃掉；若其中一个操作数是浮点数，则商的小数部分会被保留。

事实上，当开发者尝试将类型不同的变量结合运算时，C++ 把所有相关的类型都转化成相同的。

模运算符

模运算符返回整型数相除时的余数。

模运算要求两个操作数都是整型数，对浮点数使用模运算会造成编译错误。如果模运算中有一个操作数是负数，则结果的符号应该遵循如下规则：$(a\div b)\times b + a % b=a$

类型转换

C++ 在以下场景时，可自动执行类型转换：

  • 当开发者给一个算术类型变量赋予另一个算术类型的值时，C++ 对值进行类型转换
  • 当开发者在表达式中将不同类型结合起来时，C++ 对值进行类型转换
  • 给函数传递参数时，C++ 对值进行类型转换

在初始化和赋值时的转换

以上，当把类型为 short 的 s_x 赋值给类型为 long 的 l_x 时，程序会把 s_x 的值扩展为一个 long 的值。因为这种扩展创建了一个新的值放入到 l_x 中，所以原先 s_x 中的值并不会被改变。

通常来说，把一个值赋给表示范围更大的类型变量，是不会出什么问题的；但是把一个值赋给表示范围更小的类型变量的话，会造成精度的损失。还有一些其他可能出现的问题：

  • 大浮点数值赋予小浮点数类型变量：精度损失，有效数字减少
  • 浮点数值赋予整型类型变量：丢失分数部分。另外，浮点数可能太大以致于无法放入整型变量中，这时赋予结果是不确定的，也就是说，结果因环境而异。比如说，int debt = 7.2E12;。
  • 大整型数值赋予小整型类型变量：可能会发生溢出。通常来说，仅有低位的字节被复制过去了。

零值赋予布尔类型时，得 false；非零值赋予布尔类型时，得 true。

C++11 中使用 {} 初始化时的转换

C++11 称使用 {} 的初始化为列表初始化（list-initialization）。列表初始化在类型转换上更加严格。

列表初始化不允许缩小（narrowing），也就是说：类型变量可能无法表示该值时，列表初始化不允许赋值操作的完成：

  • 把整型数转换为其他整型类型或浮点类型，在编译器判断目标变量可正确存放值后，允许
  • 至于其他种类的转换，在值为该类型可处理的常量时，可能允许

表达式中的转换

C++ 在结合两种不同的算术类型于一条表达式时，会做两种自动转换：

  1. 一些类型无论如何都会被自动转换

    • 当检验表达式的值时，C++ 会把 bool、char、unsigned char、signed char 和 short 值转化为 int 值
    • 其中，bool 的值，true 转换为 1，而 false 转换为 0
    • 这些转换我们称之为整型提升（integral promotion）
    • 以上，在执行第 3 条语句时，右边两个变量的值都被转换成 int，计算得到结果后，再转换成 short 存到 fowl 中。这看起来很没有必要，但事实上，int 类型是计算机的最自然类型，也就是说，计算机可能在计算 int 类型值时最快。
    • 还有一些别的提升例子，比如说，unsgined short 在 short 比 int 小的时候，会被转换为 int；若 short 和 int 同样大，则 unsigned short 会被转换为 unsigned int。两种选择，都是为了保证在提升 unsigned short 时不会发生数据丢失。
    • 类似地，wchar_t 会被提升到以下第一个宽到足以容纳它的类型：int、unsigned int、long 和 unsigned long。

  2. 一些类型在表达式中与其他类型结合时才会被转换

    • 类型较小的值会被转换为另一更大的类型。编译器会检查一个清单，用以决定应该要在一个算术表达式中执行什么样的转换：
      1. 若其中一个类型为 long double 则另一个也转化为 long double
      2. 否则，若其中一个类型为 double 则另一个也转化为 double
      3. 否则，若其中一个类型为 float 则另一个也转化为 float
      4. 否则，若其中一个类型为整型，则执行整型提升
        1. 若两个操作数都是 unsigned 或者 signed，则低级的被转换成高级的
        2. 若两个操作数中一个 unsigned，另一个 signed：
          1. 若 unsigned 的类型比 signed 的高级，则后者转换成前者的类型
          2. 若 signed 的类型可表示 unsigned 类型的所有值，则后者转换成前者的类型
          3. 若两者等级一致，则两者都转换为 signed 类型的 unsigned 版本
    • 该清单引入了整型类型的分级概念：
      • 对于有符号整型来说，基本分级为：long long，long，int，short，和 signed char。
      • 对于无符号整型来说，就是对应的无符号版本。
      • char，signed char，unsigned char 有着同样的等级。
      • bool 处于最低等级。
      • wchart_t，chart16_t，char32_t 跟它们的基本类型处于同一等级。

在传递参数时的转换

通常情况下，C++ 函数原型控制参数传递时的类型转换。

虽然不是很恰当，但是在传递参数时舍弃函数原型控制，也是可能的。

这种情况下，C++ 对有无符号版本的 char 和 short 执行整型提升。

为了保持跟大量 classic C 代码之间的兼容性，C++ 在把 float 参数传递给违反函数原型的函数时，把它们都提升为 double 型。
