移动语义和右值引用

移动语意，move semantics。

对移动语义的需求

考虑以下使用场景，使用复制构造函数创建拷贝，和使用 allcaps 把 vstr 中的字符串都变成大写：

#1 中，其目的是创建一个新的向量，那么该语句会调用复制构造函数，程序使用 new，再动态分配空间用以存储拷贝；

#2 中，其目的是创建一个新的向量，其存放的字符串都是大写版本，在上面的调用中，可以细分为以下步骤：

  • 实参 vstr 传入 allcaps，程序使用 new 创建了全大写版本的 vstr，也就是 temp。
  • allcaps 返回语句创建了临时变量，再使用 new 创建了 temp 的拷贝，删除 temp，返回临时变量。
  • vstr_copy2 接收到返回的临时变量，调用复制构造函数，程序又一次使用 new 创建临时变量的拷贝，存放于 vstr_copy2 中。

可见，#2 中有 3 次使用 new，也就是复制了 3 次同样规模的数据，一个优秀的编译器可以把 #2 的第 2 步省去，但还是比 #1 多了一次。而细心地观察过后，我们可以发现，即便是将源向量转化成全大写版本，最直接的办法也只需要创建一次拷贝，那就是使全大写版本的向量直接成为 vstr_copy2。

概括地说，我们需要让编译器知道何时要或不要做拷贝，这就是右值引用登场的时候了。我们可以定义两个构造函数，一个是常规的复制构造函数；另一个是移动构造函数，其可使用右值引用。移动构造函数可绑定右值参数，比如说 allcaps(vstr) 的返回值，只做对象的转让工作：把 allcaps(vstr) 直接转让给 vstr_copy2。

移动构造函数可能会在转移拥有权的过程中修改其参数，所以这意味着右值引用的参数不应该有 const 限定符。

强制移动

强制使用移动构造函数，或者强制使用移动赋值运算符，其本质是强制使左值变成右值。我们可以使用 static_cast<> 运算符来强制转化对象的类型为 Useless &&。C++11 提供了一个更为简便的方法：std::move()，其在头文件 utility 中声明。

std::move() 并不是一定会产生移动操作，如果 move() 的参数对象的类没有定义移动赋值运算符，则赋值的结果是调用复制赋值运算符。如果连复制赋值运算符也没有定义的话，那么该赋值是不被允许的。

右值引用给大多数程序员带来的并不是新的代码编写方式，而是提供了使用库代码的机会，这些库代码利用了右值引用以实现移动语义。

新类特性

特殊成员函数

默认构造函数（default constructor），是不需要任何参数的构造函数。如果我们没有为类提供任何构造函数的话，编译器会提供一个默认的默认构造函数（defaulted default constructor）。该默认的默认构造函数不初始化任何 C++ 内置类型的变量，但是会调用其对象成员的默认构造函数。

如果我们没有为类提供复制构造函数且代码中的确使用到复制构造函数的话，则编译器会提供一个默认的复制构造函数（defaulted copy constructor）；如果我们没有为类提供移动构造函数且代码中的确使用到移动构造函数的话，则编译器会提供一个默认的移动构造函数（defaulted move constructor）。

类似地，编译器会提供默认的复制赋值运算符和默认的移动赋值运算符。最后，如果我们没有提供任何析构函数的知，编译器会提供默认的析构函数。

但是上面的描述是有例外的：

  • 如果我们提供了析构函数，或者是复制构造函数，或者是复制赋值运算符，则编译器不会自动提供移动构造函数，或者移动赋值运算符；
  • 如果我们提供了移动构造函数，或者是移动赋值运算符，则编译器不会自动提供复制构造函数，或者复制赋值运算符。

另外，默认的移动构造函数和默认的移动赋值运算符，跟默认的复制构造函数和默认的复制赋值运算符的行为是差不多的：

  • 对于内置的 C++ 类型，按成员初始化和复制；
  • 对于类对象，调用对应的移动（或复制）构造函数和移动（或复制）赋值运算符。

  • 简洁性：相较于 functor 需要一大串代码用于声明类，lambdas 的定义很简洁。

  • 效率：lambdas 是未命名的内联函数，其效率比一般的非内联函数高。

  • 能力：lambdas 可以访问作用域内任何自动变量。要用的变量是通过 lambdas 的中括号被捕获的。

    • 如果中括号中只是使用变量名，则该变量是通过值获取的；

    • 如果中括号中使用变量名外，还有取址符，则该变量是通过引用获取的；

    • 中括号中使用单个 &，意味着 lambdas 可通过引用访问所有自动变量；

    • 中括号中使用单个 =，意味着 lambdas 可通过值访问所有自动变量。

    • 以上情况可以自由组合。

更多 C++11 特性

并发编程

C++11 定义了一个内存模型，通过添加 thread_local 关键字，还有其他库，使其支持线程执行。

thread_local 关键字是用于声明变量拥有与线程相关的静态存储时间的，也就是说，对于用 thread_local 声明的变量，定义该变量的线程过期时，该变量也会过期。

额外的库

  • random：提供扩展后的随机数库
  • chrono：支持处理时间的方法
  • tuple：支持 tuple 模板。tuple 是 pair 对象的泛化，pair 可以持有两个类型不一定相同的值，而 tuple 对象可以持有多个不同类型的值。
  • ratio：支持编译时有理数运算的库
  • regex：支持正则表达式库。中括号表达式匹配中括号里的单个字符；\d 匹配数字，\w 匹配单词，\t 匹配制表符。因为 C++ 中的 \ 是转义字符的开始，所以 \d 等会被识别成转义字符，所以书写正则表达式的时候，需要写转义后的 backslash：\\d\\t\\w\\d。而这也是 C++ 要引入原始字符串的原因之一。在原始字符串的加持下，可以这样写上面的正则表达式：R"\d\t\w\d"。

低级编程

C++11 放松了对 POD 的限制。

C++11 使得 union 更灵活：允许其成员有构造函数和析构函数，但还是有很多限制：不允许有虚拟函数等。

C++11 解决了内存对齐问题：增加了 alignof() 运算符，还有 alignas 限定符等。

C++11 引入 constexpr 机制，允许在编译时计算出表达式的常量值，并将其存储在只读内存中；在这之前，任何变量都是在运行时计算，存储在随机访问内存中。

杂项

C++11 允许扩展整型类型。扩展类型由头文件 cstdint 支持。

C++11 提供文字运算符（literal operator），像 1001001b 这样的字面常量可以用于表示二进制数，被转化为整型值。

C++11 添加了 static_assert 关键字，可以在编译时对断言进行测试。在这之前，C++ 的调试工具 assert 是在运行时检测断言的，如果断言为 false 的话，则展示信息，且调用 abort()。

C++11 对元程序设计（metaprogramming）提供了更多支持，元程序设计旨在创建“可以创建或者修改其他程序的”程序。
