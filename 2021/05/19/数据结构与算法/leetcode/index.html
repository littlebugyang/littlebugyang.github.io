204. 计数质数

204. 计数质数

如何理解埃式筛从 $x\cdot x$ 开始标记的做法：

  • 从 $x\cdot x$ 开始标记并不是只标记 $x$ 的幂的位置。
  • 在 $x\cdot x$ 之前，一定有比 $x$ 更小的质数把该更小质数的所有倍数位置都标记了。
  • 而这个比 $x\cdot x$ 小，却比 $x$ 大（起码 1 倍以上，因为标记为合数的不包括 $x$ 本身）的合数的质因数，除了 $x$ 之外，一定还有其他的质数，因为每个合数都可以写成几个质数相乘的形式，参考：https://baike.baidu.com/item/%E8%B4%A8%E5%9B%A0%E6%95%B0/6192269
  • 对于“一定有比 $x$ 更小的质数把该更小质数的所有倍数位置都标记了”这个说法，2 是例外，因为在 2 之前，并没有其他的合数。好在 2 和 2x2 之间的 3 本来就是质数，所以从 $x\cdot x$ 开始标记，对于 $x=2$ 来说，并没有漏掉任何合数。

但是在我动手去写的时候，我才发现，原来是不需要计算该数是否为质数的：所有的合数都有质因数，所以从 2 开始，对每个质数的倍数都标记为合数，则可以覆盖所有的合数。

另外，还有处理 0、1 的情况，我刚开始的时候，为了极致地减少使用空间，只申请了 n-2 的大小的 vector，所以，当 n 为 0 或 1 的时候，申请 n-2 的大小的 vector 就会抛出异常。

总之，在来回对比中，我的代码就完全变成了 leetcode 官解的模样。

28. 实现 strStr()

28. 实现 strStr()

我想借这个机会自己实现 KMP 算法，没想到，一上来就掉坑了：

在 strStr() 函数中，刚开始我打算少声明几个变量，直接用 (i < haystack.size() && j < needle.size()) 作为 while 循环的条件语句，但是没有想到这里也能翻车。

因为 size() 返回的是无符号类型，而其与 i/j 比较时，如果 i/j 为负数，比如说 -1，则该 -1 会被转化为无符号数，再与 size() 们比较；而 -1 转化为无符号数会变得很大很大……这道题中一开始，j 就要小于 0（对应首次对比失败的情况），所以循环只会执行一次，这就使得程序无法正常结束。

难怪之前遍历向量的时候，编译器总是不让我用 int i = 0，改成 unsigned int i = 0 才肯罢休，原来是这个原因……

这首题不能够一开始就把 i/j 声明成 unsigned int，因为 j 是有小于 0 的时候的。
