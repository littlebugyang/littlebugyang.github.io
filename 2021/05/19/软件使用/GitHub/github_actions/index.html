软件使用►GitHub

github_actions

概述

GitHub Actions 是 GitHub 推出用于的 CI 工具。

参考

https://docs.github.com/en/actions

GitHub 的 docs 真的啥都有，但是就是中文不走心，居然有翻一半剩一半的……

示例

反正是为了使用 hexo 而学的 GitHub Actions，所以以 hexo 提供的 .yml 来学习一下吧：

上面用到了几个外部动作代码库：

  • actions/checkout@v2
  • actions/setup-node@v1
  • actions/cache@v2
  • peaceiris/actions-gh-pages@v3

其中 @ 后面是用于指示代码版本的。在 uses 的文档里有提到。

uses: checkout

关于 checkout 这个动作，其文档中称：该动作是用来在 $GITHUB_WORKSPACE 下 check-out 我的仓库的，这样 workflow 才能访问该仓库。看得我是一头雾水。再结合 $GITHUB_WORKSPACE 的解释，我大概能明白了：check-out 仓库后，仓库才会被复制 GitHub 工作区，也就是在这个工作区中，发生安装包、构建包、部署等一系列步骤。

uses: setup-node

顾名思义，该动作设立了 node 环境。

我看了一下 setup-node 的主页，发现 v2 都出了，不知道 hexo 的文档是没更新还是咋样，但是 v2 是去年 10 月就出了，所以我不是很相信是为了稳定才没选 v2。

在该动作后跟着 with 属性，根据其文档，该属性是用于设置环境变量的。所以，node-version 应该成为了一个值为 12.x 的环境变量。这样一来，with 倒成了一种输入了，而文档中也称 node-version 为可选的输入。

uses: cache

This action allows caching dependencies and build outputs to improve workflow execution time.

又可以顾名思义了。

uses: actions-gh-pages

专门用来部署 GitHub Pages，除了 publish_dir 和 publish_branch，关于 token 我暂时还没有搞懂。

run

Runs command-line programs using the operating system’s shell.

总结

如果使用该配置文件部署博客，则需要将 public 文件夹保持 ignored，平时在 source 分支上编辑代码。
