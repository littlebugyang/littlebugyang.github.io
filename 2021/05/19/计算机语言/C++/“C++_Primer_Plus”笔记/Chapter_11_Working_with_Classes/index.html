重载限制

  1. 大多数运算符都可以重载：
    1. +, -, *, /, %, ^
    2. &, |, ~, !, =, <, >
    3. +=, -=, *=, /=, %=, ^=, &=, |=
    4. <<, >>, >>=, <<=, ==, !=, <=, >=
    5. &&, ||, ++, --, ,, ->*, ->, ()
    6. [], new, delete, new [], delete []
    7. 以下的运算符重载时必须定义为成员函数：=, (), [], ->
  2. 不能够重载以下运算符：
    1. ., .*, ::, ?:
    2. typeid, const_cast, dynamic_cast, reinterpret_cast, staic_cast
  3. 被重载的运算符不是一定要为成员函数。
  4. 至少有一个操作数是用户自定义的类型。这是为了避免重载标准类型的运算符。
  5. 不能够违反操作符原来的语法规则。比如说，不能够把 % 运算符重载为只需要一个操作数。
  6. 不能修改运算符优先级。
  7. 不能添加新的运算符标志。
  8. 除了以上这些硬限制，在重载运算符函数时，我们应尽量把运算符的标志和实际操作联系起来。

重载运算符：成员函数 VS 非成员函数

非成员函数一般来说需要为类的友元函数。

使用成员函数的话，一个参数可以隐式地通过 this 指针使用；而非成员函数中，所有操作数都必须作为参数被传入。

如果同时定义了非成员函数和成员函数的运算符重载，且两者都匹配了同一表达式，那么会产生编译错误。

那么最好是使用哪种呢？对于某些运算符，是只能作为成员函数的；对于其他的运算符，其实哪种形式的重载都无所谓；对于某些特定的类设计来说，非成员函数的版本有一个优势，尤其是在为该类定义了类型转换的时候（我没看懂）。

如果一个方法要重新计算出一个新的类对象的话，那么就很有必要看看构造函数是否能胜任这份工作。这不仅可以省去很多麻烦，而且这可以确保新对象是按照当初设想那样构建的。

再提炼：重载已重载运算符

给 Vector 类重载运算符，仅使用一个负号起取反作用的时候，编译器并不是把“对象-”当作匹配的表达式，而是匹配“-对象”，这与重载双目运算符时不一样。

因为运算符的重载是通过函数实现的，所以运算符可以一直被重载，只要每个运算符函数都有独一无二的函数签名，而且运算符需要的操作数个数与内置时的一置即可。也就是说，“/”所需要的操作数是两个，那么重载的时候也必须是需要两个操作数。

让 Vector 类随机漫步

讲到酒鬼问题：一个喝醉酒的人，需要走多少步才能够到达距离出发点特定长度的点。

平均来说，设置步长为 s，距离为 D，步数为 N，其关系为：

$$N = (D/s)^2$$

但这只是一个平均估计，实际上，具体结果会围绕理论值有很大的浮动。当尝试的次数接近无限大的时候，才能得出与理论值相近的值。

关于随机函数

rand() 函数其实是伪随机的，因为通常来说，每 10 个连续的调用会产生同样的 10 个结果，具体的数值取决于实现环境。srand() 可以覆写默认的种子值，初始化一个不同序列的随机数。在酒鬼问题的代码中，使用了 time(0) 的返回值作为 srand() 的参数，也就是种子。time(0) 通常返回当前日期距离某特定日期的时间，单位为秒。一般来说，time() 把 time_t 变量的地址作为参数，但是使用 0 的话，可以避免使用 time_t 这个类型。
