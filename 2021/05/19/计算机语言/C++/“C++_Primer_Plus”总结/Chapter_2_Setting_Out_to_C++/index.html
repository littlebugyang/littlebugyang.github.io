头文件

像 iostream 这样的文件被称作包含文件（include file）或者头文件（header file），因为它们总是在文件的头部被包含进程序里。

在 C 中，也可以使用 iostream 的功能，只不过要包含的头文件变成了 iostream.h。

现在，后缀 .h 的头文件被保留为 C 的头文件，C++ 程序也可以使用 C-Style 的头文件；而 C++ 则使用没有后缀的头文件。

原有的 C 的头文件，在去掉 .h 的后缀后，再在名字前面加上“c”（来自 C 语言），就成了符合 C++ 标准命名的头文件。比如说：cmath

注释

C++ 注释以 // 开头，这是在 C++ 中引入的注释方法。

C 语言注释以 /* ... */ 为界；C++ 也允许这种注释方法。

在 C99 中，单行注释 // 也被引入到 C 语言中。

语句

在 C++ 中，每条完整指令（instruction）被称作语句（statement）。

终结符

每个语句必须以分号（semicolon） ; 终结。

C++ 中，; 为终结符（terminator）。

分割符

与终结符类似的，还有分割符（separator）。

在 Pascal 语言中，编译器以分号分割语句。在某些特定场景下，程序员还可以去掉分号。因为分号在 Pascal 中只是一个分割符，当后面没有需要再区分为另一条语句的字符时，便不再需要分号。

在 FORTRAN 语言中，编译器以回车（carriage return）分割语句，即：一行即为一条语句。

终结符和分割符

终结符和分割符两者的区别是：前者是语句的一部分，而后者只是语句之间的标记（marker）。

这导致的结果是：在 C/C++ 的源码中，语句的分号在任何情况下都必须保留。

而且，C++ 把源代码中的回车当作空格或制表符，也就是说，在 C++ 的源代码中，空格和回车可以互换。这同时也意味着，C++ 可以把一个语句分成好几行，或者把好几个语句放在同一行。

标记

不可分割的元素为标记（token）。通常来说，标记之间必须有空白符（white space），也就是空格、制表符或者回车。单字符的标记如括号和逗号等，是不需要用空白符分隔的。

声明语句

在计算机中存储信息，必须要做的是：明确存储位置和存储大小。在 C++ 中，用声明语句（declaration statement）就可以做到明确存储大小和提供访问位置的标签。

那么为什么变量必须要被声明呢？在一些语言中，变量是不需要声明的，要用的时候直接写就行了。但是当我们写错变量名的时候，不带声明的变量使用，很容易让人忽略拼写错误。

C++ 中，只要在使用变量前，声明该变量即可，而不需要像一些其他语言那样，提前把所有变量都声明好。

函数

int main() 为函数头部（function header），在大括号（braces） {} 内部的为函数体（function body）。

{} 在 ISO 标准中被称作“braces”。

通常情况下，C++ 的函数是被另一个函数所激活的，或者说被调用的（called）。

被调用函数的头部包含了调用函数（calling function）与被调用函数（called function）之间所使用的接口：

  • 函数返回类型（function return type）：描述信息类型，该信息从被调用函数返回给调用函数
  • 参数列表（argument list or parameter list）：描述信息个数和类型，此信息从调用函数流向被调用函数

main 函数

main 函数也是被调用的，调用者为 startup code，该段代码是编译器添加的，作为程序与操作系统的中介。

返回语句

ISO C++ 标准给部分不想在 main 函数最后加上 return 0; 语句的开发者们让步，规定：即便在编译器到达 main 函数末尾处之前都没有遇到返回语句，编译器也要能够保证最终效果与人工写上 return 0; 的一致。这个规定仅限于 main 函数。

名字

为什么是 main 函数，而不是 Main 函数或其他的？应该仅仅是规定。

作用

运行 C++ 程序的时候，执行操作总是从 main 函数开始。如果源代码没有 main 函数的话，编译器会报错。

但是这也有例外。比如说：

  • 在 Windows 下编程会用到动态链接库（dynamic link library，DLL）模块，这些模块包含了 Windows 程序能用的代码。DLL 模块不是一个单独程序，所以它们是不需要 main 函数的。
  • 在特殊环境下编程，比如说给机器人的控制芯片编程时，可能不需要 main 函数
  • 在某些特殊环境下，编程环境提供的程序框架调用非标准的函数，比如说 _tmain();，在这种情况下，实际上是有一个隐藏的 main 函数调用了 _tmain() 的。

函数原型仅仅定义了函数的接口，与函数定义是不同的。

在一些其他的编程语言中，function 这个术语是专门给有返回值的函数的，而把那些没有返回值的函数称作 procedure 或者是 subroutine。但是在 C++ 中，这两种函数，都称作 function。

main 函数也是有返回值的，其实可以看作是操作系统在调用程序的 main 函数，所以 main 函数的返回值并不是返回给程序中的哪一部分，而是返回给操作系统。许多操作系统可以利用返回值作为退出值（exit value），判断程序是否正常退出。通常来说，0 返回值代表程序成功运行，非 0 值代表程序在运行时出现了问题。

另外，main 函数看似一个预留的关键字，但事实上 main 不是关键字，因为这不是编程语言的一部分，这只是一个必须的函数名。但是开发者最好还是不要用 main 来命名变量。

endl

endl 在使用的过程中，跟“\n”在显示效果上没有什么区别。

但是 endl 有一点不同，endl除了写“\n”进外，还调用 flush 函数，刷新缓冲区，把缓冲区里的数据写入文件或屏幕。

也就是说，使用 endl 可以保证换行效果立刻显示在屏幕上；而当使用“\n”时，系统可能先把“\n”暂存在缓冲区中，等待时机再输出。

对类的一点接触

类（class）是用户自定义的数据类型。

之前我们把 carrots 定义为 int 类型，在这种情况下，carrots 有着 int 类型的属性；常用的 cout，在 iostream 中被定义为 ostream 类的对象，有着 ostream 类型的属性；类似地，cin 的具有 istream 类的属性的对象。

关于 ostream 和 istream 的定义，是写在 iostream 里的，而不在编译器里，所以，我们甚至可以修改 iostream 的内容，但是这是很危险的。
