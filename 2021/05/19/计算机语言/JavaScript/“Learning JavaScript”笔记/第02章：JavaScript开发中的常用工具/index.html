模块依赖

npm 通过一个名为 package.json 的文件来帮助追踪项目所需要安装模块的依赖关系。在项目目录下运行命令就可以创建该文件：

依赖分为：常规依赖（regular dependencies）和开发依赖（dev dependencies）。

前者是应用运行必需的，而后者只是在构建项目过程中有用或者必需的。

安装依赖时，加上 --save 或者 --save-dev 的标记，能把该安装包的信息加入到 package.json 当中；否则，npm install underscore 这样的语句只是安装了软件包，并没有使其变得被这个项目所依赖。

而对于 –save 和 –save-dev，前者对应常规依赖，后者对应开发依赖，在参考的 stackoverflow 的答案中，有个例子讲得很清楚。
